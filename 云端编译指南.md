# 云端编译 APK 指南（无需安装 Android Studio）

> 由于当前网络无法访问 Google 的 Android SDK 下载服务器（dl.google.com），本地编译可能失败。
> 本指南介绍如何通过 GitHub 在云端编译 APK，完全不需要在本地安装 Android SDK。

---

## 前提条件

- 一个 GitHub 账号（免费注册：https://github.com/signup）
- 浏览器（Chrome、Edge 等）

## 方法一：通过 GitHub 网页上传（最简单，无需安装任何软件）

### 第1步：注册 GitHub 账号

1. 打开 https://github.com/signup
2. 输入邮箱、密码、用户名
3. 完成验证
4. 点击 "Create account"

### 第2步：创建新仓库

1. 登录 GitHub 后，点击右上角 **+** → **New repository**
2. Repository name 填写：`kidsguardian`
3. 选择 **Public**（公开，这样 GitHub Actions 免费）
4. 勾选 **Add a README file**
5. 点击 **Create repository**

### 第3步：上传项目文件

1. 在仓库页面，点击 **Add file** → **Upload files**
2. 将项目目录 `d:\AliWangWang\少儿卫士` 中的以下文件/文件夹拖入：
   - `app/` 文件夹（包含源代码）
   - `build.gradle`
   - `settings.gradle`
   - `gradle.properties`
   - `gradle/wrapper/gradle-wrapper.properties`
   - `.github/workflows/build.yml`
   - `.gitignore`

3. 注意：不要上传 `local.properties` 文件（已包含在 .gitignore 中）

4. 在页面底部填写 commit message：`上传少儿卫士项目`
5. 点击 **Commit changes**

### 第4步：创建 Gradle Wrapper 文件

GitHub Actions 需要 `gradlew` 和 `gradlew.bat` 脚本以及 `gradle-wrapper.jar`。
请将以下内容创建为文件并上传：

**文件路径：** `gradle/wrapper/gradle-wrapper.properties`（已创建）

**注意：** `gradlew`、`gradlew.bat` 和 `gradle-wrapper.jar` 需要特殊处理。
GitHub Actions 会自动使用 setup-android action 来处理。

### 第5步：等待自动编译

1. 上传文件后，点击仓库上方的 **Actions** 标签
2. 你会看到一个名为 "Build APK" 的 workflow 正在运行
3. 等待约 5-10 分钟（绿色对勾表示成功）

### 第6步：下载 APK

1. 点击成功的 workflow run
2. 在页面底部找到 **Artifacts** 区域
3. 点击 **kidsguardian-debug-apk** 下载
4. 解压下载的 zip 文件，里面就是 `app-debug.apk`

### 第7步：安装到手机

1. 将 APK 文件传到手机
2. 在手机上点击安装
3. 如果提示"未知来源"，按提示允许安装

## 方法二：通过 Git 命令行上传（如果方法一上传文件较多时有困难）

### 安装 Git

PortableGit 文件已下载到 `d:\AliWangWang\PortableGit.exe`
双击运行，解压到 `d:\AliWangWang\PortableGit`

### 配置 Git

打开 PowerShell，执行：
```powershell
$d = "d:\AliWangWang\PortableGit"
$env:PATH = "$d\cmd;$d\mingw64\bin;$d\usr\bin;$env:PATH"
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

### 初始化并推送

```powershell
cd "d:\AliWangWang\少儿卫士"
git init
git add .
git commit -m "初始提交：少儿卫士项目"
git branch -M main
git remote add origin https://github.com/你的用户名/kidsguardian.git
git push -u origin main
```

输入 GitHub 用户名和 Personal Access Token 作为密码。

## 常见问题

### Q: Actions 运行失败怎么办？
A: 点击失败的 run，查看日志。常见问题：
- 缺少 gradlew 文件：在仓库中添加 gradlew 脚本
- 编译错误：检查代码是否有语法问题

### Q: 如何获取 Personal Access Token？
A: GitHub → Settings → Developer settings → Personal access tokens → Generate new token，勾选 `repo` 权限。

### Q: Artifacts 下载链接过期了怎么办？
A: 重新触发 workflow：Actions → Build APK → Run workflow
